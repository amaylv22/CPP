Objetos constantes (de clases) y funciones miembro

Un objeto de determinada clase declarado como constante (CONST [NAME_OBJ]) sirve para garantizar que no se hagan modificaciones sobre los valores en él almacenados.

Los objetos const deben ser inicializados en una declaración, si no hay un constructor con parámetros se debe de proporcionar un constructor público por defecto para inicializar las variables. De otro modo genera error.

Si se intenta modificar el valor de las variables de un objeto const, este genera un conflicto, además los objetos const solo pueden llamar a funciones const.

Función const.

Es una función que garantiza que no se harán cambios en ninguna variable y que se llenen con funciones que no sean const.

<TIPO> [NOM_FUNCION] (<ARGUM>...) CONST {...}

Desde que el constructor no esté mencionado como función const, es aquí donde se deben inicializar las variables.

Es posible sobrecargar una función para que tenga una (...) CONST y otra no CONST, así la variable CONST solo será llamada en las declaraciones como tal.
