Destructores

Los destructores son funciones miembro especiales que sirven para eliminar un objeto de una determinada clase, liberando la memoria utilizada por dicho objeto. Así pues, limpian el objeto y liberan secciones de memoria.

Características de un destructor.

  • Se declaran siempre que se declara un constructor.
  • No tienen valor de retorno.
  • No toman argumentos.
  • No pueden ser heredados.
  • Tienen el mismo nombre de la clase, pero los antecede el símbolo (~). (alt + Ñ + |ESPACIO|)
  • No incluye acciones.

Siempre que se define un destructor para una clase, este es llamado automáticamente cuando se abandona el ámbito en el que fue definido. Esto es así solo cuando el objeto fue creado dinámicamente con el operador (NEW), ya que en este caso si es necesario eliminarlo con el operador (DELETE).

El destructor no puede sobrecargarse.

Declarar el destructor.

~ %ltNOM CLASE%gt ( );

Definir el destructor.

<NOM CLASE> :: ~<NOM CLASE> ( )
	{
		<SENTENCIA>
	}

<SENTENCIAS O ACCIONES> En el caso de tener datos miembro tipo puntero se aconseja incluir acción en el destructor que se encarga de liberar la memoria ocupada por dado puntero, en caso contrario la definición incluirá solo las llaves.

<NOM_CLASE> :: ~<NOM CLASE> ( )
	{
	DELETE <PUNTERO_NAME>;
	.
	.
	.
	}

DELETE <PUNTERO_NAME>; es para cada uno de los datos miembro tipo puntero en memoria Lead, es decir, inicializados con new.
