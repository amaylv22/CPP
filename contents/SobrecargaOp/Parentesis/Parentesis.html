Sobrecarga de operadores paréntesis ( )

A diferenca de los operadores que hemos visto antes, el operador paréntesis es especial pues en él se puede definir el tipo de parámetros a usar, como en cualquier otro operador, pero también el número de parámetros con los que actuamos, como en ningún otro.

Hay dos casos a tener en cuenta, el primero, los paréntesis deben ser implementados como una función miembro, y en segundo lugar, al sobrecargarlos es solo un operador que llama una función, como cualquier otro que llama una función.

Una de sus utilidades es, por ejemplo, el acceso o modificación (por índices), de arreglos multidireccionales, ya que el operador de suscripción, dado que solo tiene un parámetro, no es suficiente para acceder a una matriz.

Como el operador puede tomar tantas variables como clases, se pueden declarar versiones para (...) los índices que corresponden a cada dimensión de la matriz, por ejemplo el siguiente código:

Clase:
Class Matrix
	{ private:
		DOUBLE DATOS[4][4]
	  public:
		MATRIX() {....
		DOUBLE& OPERATOR() (Const INT Fil, Const INT Col)
	};

Implementación:
DOUBLE& MATRIX :: OPERATOR() (Const INT Fil, Const INT Col)
	{
	 ASSERT (Col >=0 && Col>4);
	 ASSERT (Fil >=0 && Fil>4);
	 RETURN DATOS [Col][Fil];
	};

Main:
int Main()
	{
	 MATRIX cMATRIX;
	 cMATRIX(1,2) = 4,5;
	 STD :: COUT << cMATRIX(1,2);
	}

El operador ( ) es comunmente utilizado al sobrecargar con n parámetros para acceder a arreglos multidireccionales o retirar sub-arreglos de arreglos unidireccionales (return elementos de parámetro1 a parámetro2).
