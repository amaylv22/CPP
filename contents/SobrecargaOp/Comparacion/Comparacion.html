Sobrecarga de operadores de comparación (==, !=, <...)

Los operadores de comparación son también binarios, así que se utilizan funciones amigas para hacer su sobrecarga. Es importante recordar que los datos de salida, los resultados arrojados por estos operadores son booleanos.

Hablando sobre las operaciones de igualdad o desigualdad (= = y !=), es importante hacer nota que != obtiene la respuesta opuesta que = =, es decir, si al usar = = obtenemos true, al usar != obtendremos false y viceverza, por tanto podemos usar una en función de otra, generalmente != en función de = =, lo que hace las cosas simples y además reduce la posibilidad de errores y la cantidad de código a escribir. Tal como se muestra en el siguiente ejemplo:

bool OPERATOR == (CLASS A &P1, CLASS A &P2)
{
RETURN (P1.x==P2x && P1.y==P2.y && P1.z==P2.z);
}

bool OPERATOR != (CLASS A &P1, CLASS A &P2)
{
RETURN ! (P1==P2)
}

La misma forma de sobrecargar puede ser usada para los demás operadores de comparación (<, >, <=, >=).

Nótese también que los operadores > y <= son lógicamente opuestos también, al igual que < y >=, es decir, uno puede ser definido en términos de otro.
