Sobrecarga de operadores suscritos ([ ])

Cuando se trabaje con arreglos, generalmente se utiliza el operador suscrito ([ ]) para acceder por medio del índice a tantos espacios del array.

Ahora, si en una clase tenemos una variable de tipo array privado, ¿de qué manera/forma podremos acceder a los datos del array o modificar los datos del mismo, si este es privado? La forma indicada será crear un par de funciones públicas, una para leer a la que habría que proporcionarle el índice de consulta y otra para escribir a la que hay que darle elíndice y el dato a acceder, sin embargo eso genera una situación no amigable al usuario. Lo mejor en este caso es sobrecargar el operador suscrito ([ ]).

Como sabemos, el operador [ ] toma como argumento un dato entero [i], patra que sea su valor de retorno, veamos que es lo que debemos de hacer.

  1. Visualizar el contenido del índice que se le indica.
  2. Asegurar un valor a la (...) del índice que se le indica. Este valor es del tipo del que fue declarado el arreglo.

Para cumplir ambas funciones, la sobrecarga debe regresar una referencia, pues de este modo al usar el operador de asignación (=) se le puede asignar un valor a la referencia y este puede ser consultado normalmente como si se tratara de un valor de una variable normal.

La referencia debe ser a algún tipo de variable, este <TIPO> será igual al tipo con que se ha declarado el array.

<TIPO>& OPERATOR[ ] (INT 1)
	{
	RETURN MemberArray[i]
	}

Además, al sobrecargar [ ] tenemos la ventaja de que, debido a que necesitamos diferentes funciones, reocuparse el índice es válido, o incluso usar el índice para acceder a un término específico de un string, manejándolo como un caso de char.
