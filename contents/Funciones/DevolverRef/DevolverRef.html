Funciones que devuelven referencias

Existen también funciones que devuelven referencias, para ello es necesario declarar el valor de retorno como una referencia.

<TIPO> & <IDENTIFICADOR_FUNCION> (<LISTA_DE_PARAMETROS>)

Esto nos permite que una llamada a función se comporta como un objeto, ya que una referencia se comporta como tal, la que brinda la oportunidad de hacer cosas como asignaciones con los llamados o también operaciones.

El operando "&" solamente se incluye al crear el prototipo y la definición de la función, al hacer los llamados este se debe de omitir. Veáse por ejemplo:

ESTRUCT ARRAY 25
{ INT AVALUE [25];};

dnt& VALUE (ARRAY 25 & ARRAY, int INDEX)
{ RETURN ARRAY.AVALUE[INDEX];}

En MAIN
	VALUE (MARRAY,7)=5 → Regresa la int a la componente 7 del array y la iguala.
