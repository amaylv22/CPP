"Casting", conversiones explícitas de tipos

Cuando se produce demación de tipos, el computador normalmente emite avisos o "warning", por ejemplo, al asignar expresiones tipo float o variables int.

Para aludir estos avisos, el compilador usa el "casting" o conversión explícito (y no implícito como en el caso de la demación). En general el uso de este recurso es obligatorio cuando se hacen asignaciones, o cuando se pasan argumentos a funciones con pérdida de precisión. En el caso de los argumentos pasados o funciones es también muy recomendable aunque no exista pérdida de precisión.

En general un "casting" tiene dos notaciones, la normal y la funcional, que se presenta a continuación respectivamente.

((TIPO))(VAR_NAME) o (TIPO)((EXPRESION))

Existen otros castings propios de C++, los cuales son más recomendables debido a que realizan tareas especializadas, a continuación se presenta un listado de ellos.

Static_cast< >

Realizan una conversión de tipo durante la compilación del programa, de modo que no crea más código durante la ejecución. Es obligado a realizar conversiones de tipo que de otro modo haría el compilador automáticamente,por ejemplo, convertir un puntero a un objeto de una clase determinada o un puntero a una clase base pública.

STATIC_CAST(TIPO) ((EXPRESION));

Const_cast< >

Se usa para liminar o añadir los modificadores const y volatile de una expresión. Para eso, la expresión debe de ser del mismo tipo del "(TIPO)" salvo por los modificadores const o volatile que tengan que aparecer o desaparecer.

CONST_CAST(TIPO) ((EXPRESION));

Reinterpret_cast< >

Se usa para hacer cambios de tipo a nivel de bits, es decir, el valor de la "expresión" se interpreta como si fuese un objeto del tipo "(TIPO)". Los modificadores const y volatile no se modifican, permanecen igual que en el valor asignado de "expresión". Este tipo de conversión es peligrosa.

REINTERPRET_CAST(TIPO) ((EXPRESION));
