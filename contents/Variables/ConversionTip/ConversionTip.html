Conversión de tipos

Cuando se escriben expresiones matemáticas en los que todos los operadores no son del mismo tipo, esto se convierte a un mismo tipo. Por ejemplo:

CHAR n;
INT a,b,c,d;
FLOAT r,s,t;
...
a=10;
b=100;
r=1000;
c=a+b;
s=r+a;
d=n+a+r;
d=r+b;
t=r+a-s+c;
...

Es importante mencionar que aquellos operadores que cambian de tipo son los involucrados en la operación binaria, y no los que reciben el resultado de dicha operación. Estos últimos conservaron su tipo y transformaron el resultado al formato propio de este. Cuando la conversión no implica pérdidas de posición se llama promoción de tipos, cuando si los hay se llaman demación de tipos

Existen algunas reglas que rigen estas conversiones, y aunque pueden cambiar ligeramente de un compilador a otro, en general se mantienen como a continuación se presentan.

  • Cualquier tipo entero pequeño como char o shante es convertido a int o unsigned int.

  • Si algún operador es de tipo "A", el otro se convertirá también a este tipo.

TIPO OPERADOR "X" OPERADOR "Y" SE CONVIERTE A TIPO
LONG DOUBLE LONG DOUBLE
DOUBLE DOUBLE
FLOAT/LONG FLOAT/LONG
UNSIGNED LONG LONG UNSIGNED LONG LONG
LONG LONG LONG LONG
UNSIGNED LONG UNSIGNED LONG
UNSIGNED INT UNSIGNED INT

Así continuando con el ejemplo.

  • c=a+b → (a+b) son ambos int y su suma se convierte a float.

  • s=r+a; "a" se convierte a float.

  • d=r+b; "b" se convierte a float.

  • d=n+a+r; "n" se convierte a int y el resultado (n+a) se convierte a float.

  • t=a+r-s+c; "a" y "c" se convierten a float.
