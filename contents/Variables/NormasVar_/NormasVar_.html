Normas de notación para el contenido de las variables

Constante Int

Se puede almacenar en tres sistemas diferentes, hexadecimal, octal y decimal. La ventaja es que para números enteros en un byte se pueden representar en hexadecimal rangos de 00 a ff y en dos bytes de 0000 a ffff, además de que la conversión de octal y hexadecimal a binaria es casi directa. Para definir un valor en estos sistemas se debe escribir:

  • VAR = 124 → Decimal

  • VAR = O321 → Octal (Prefijo O)

  • VAR = Ox4b → Hexadecimal (Prefijo Ox)

Especificaciones de tipos

Cuando asignamos un valor constante a una variable es necesario especificar si es de un tipo especial, con el fin de que el compilador asigne el valor de esa manera, es decir, correspondiente al tipo y este proceso no tenga que ser realizado después directamente por el programa ya en tiempo de ejecución. Esta especificación se lleva a cabo agregando sufijos al valor asignado como a continuación se muestra.

  • Ctes. Long → x = 123 L

  • Long Long → x = 123 LL

  • Unsigned → x = 123 U

  • Unsigned Long → x = 123 UL

  • Float → x = 123 F

  • Double → x = 123

  • Long Double → x = 123L (una vez declarado double)

Se pueden cambiar los prefijos "O" y "Ox" con los sufijos vistos anteriormente.

Constante Punto Flotante

Cuando no se use el prefijo (es decir, double), se debe de añadir en caso necesario (si se presenta un entero) un ".0" para indicar que se trata de una variable tipo "double".

También se puede utilizar notación exponencial:

x = 123e4

4→valor del exponente

e → exponencial

x = 123E4

Constantes Char

El valor de las constantes se representa entre comillas simples si se trata de un solo caracter, y en caso de ser más de uno, se utilizarán comillas dobles.

Existen caracteres especiales que nos (...) encerrados en comillas para representarlo, así que cuentan con una definición especial, estos caracteres son llamados "secuencias de escape". La sintaxis para representarlos es:

VCHAR ='<$ESCAPE>'
